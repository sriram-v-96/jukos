JUKOS Universal Knowledge Superset

Design Specification v1.0 · the taxonomy foundation of JUKOS · authored for implementation on the existing J0/J1 machinery
Venkat / JSAI-JeevesAI (proprietary) · 2026-07-08 · target for CC reconciliation + build
STATUS: DESIGN DECISION — to be reconciled against the corpus, then implemented before J2
The principle. JUKOS is an Operating System, not an Application. Its taxonomy is therefore a Universal Knowledge Superset — a single, comprehensive, standard-grounded structure into which all content is authored top-down, so that compliance with global standards is structural (by construction) rather than verified after the fact, and the downstream AI agent layer maps into a known, born-structured knowledge space rather than reconstructing structure at inference time.

1 · The four defining properties

The superset is defined by four properties, each of which was a deliberate design conclusion. All four hold simultaneously; none is optional.

  1. Superset, not monolith. It is the union of all relevant global domain-standards, not any single standard. There is no one global standard spanning all domains (medical has SNOMED CT / ICD; K-12 physics, JEE, and general knowledge have their own frameworks and none is universal) — so the superset comprehends all of them. The multiplicity of standards is the reason it is a superset; a single standard would be a monolithic block, which this explicitly is not.
  2. Grounded, not invented. Each domain branch is grounded in that domain's real, recognised global standard(s) — JUKOS does not reinvent SNOMED or a curriculum framework; it incorporates them as branches. What JUKOS authors is the connective upper structure that unifies these heterogeneous standards into one coherent superset. JUKOS owns the unification and the connective tissue; the branches are the real standards.
  3. Top-down derivation (born structured, compliant by construction). Content is authored by deriving downward through the superset — entering via its domain's standard-grounded branch and specialising toward the concrete lesson. Because content is born inside the structured superset, it cannot be non-compliant or unstructured: compliance and structure are preconditions of authoring, not properties to be checked afterward. Governance is by process, not by tracking.
  4. Dynamic and open (governed growth). The superset expands over time — new knowledge, inventions, research findings, and progressive changes in pedagogy extend it; new domains are added as branches. It is a living structure, not a frozen reference. Crucially, it grows only through the governed, versioned, provenance-sealed extension process (§5), so growth preserves the compliance-by-construction guarantee rather than eroding it.

Three tiers, top to bottom, mapping onto JUKOS's existing UKG→EKG→Curriculum specialisation:

  • Connective upper structure (UKG level). The JUKOS-owned, domain-neutral structure that unifies the domain branches into one coherent superset. This is the novel, high-value taxonomy IP — the "glue" that makes heterogeneous global standards derivable through one common frame. Sovereign JUKOS identity (own stable IDs) with each node bound to its standard grounding.
  • Domain branches (grounded). Each domain enters through its real global standard(s), incorporated as the branch's grounding — SNOMED CT / ICD / FMA / UMLS for medical; curriculum/syllabus frameworks for education; and so on. New domains attach as new branches (§5).
  • Downward specialisation (EKG → Curriculum → JKDL). Educational framing, then curriculum, then the concrete lesson — each a monotonic specialisation of the tier above. Proprietary taxonomies (a specific board, a syllabus, JS LDS's topic_master) bind at the Curriculum tier as content, via the adopter pattern — never as new mechanism.

3 · Top-down derivation — the enforcement

This is the property that distinguishes the superset from a mere reference catalogue, and it is the primary posture change from the corpus's current design (§6).

The rule: a knowledge node cannot be authored except by deriving downward from a parent in the superset. At mint time, the derivation is mandatory and immutably recorded — every node carries, as a required (non-null) birth binding, the superset parent it specialises and (where the branch is standard-grounded) the external standard code it is grounded in. There is no authoring path that produces a node outside the superset.

  • Standard-grounded where a standard exists. In domains with a real global standard, downward derivation terminates in a standard-grounded node — the content is provably standard-compliant because it was minted from the standard-grounded branch.
  • Structured-native where no standard exists. Where a domain (or a novel topic) has no external standard to ground in, the node is still authored into the canonical superset structure (so the agent layer still sees a known shape) — it is "structured-native": it follows the superset's structure and derivation discipline, but its grounding is a JUKOS-native superset node rather than an external standard code. The structure is always mandatory; the external-standard parent is mandatory only where a standard is available.
  • Sovereign identity, mandatory binding. JUKOS owns its internal node identities (stable, licence-clean, air-gap-safe — a hard requirement for sovereign/air-gapped deployment). The external standard is bound at birth, not substituted for identity. This preserves both compliance-by-construction (mandatory downward derivation) and sovereignty (own IDs) — the two are not in tension when the binding is mandatory-at-mint rather than optional-map-outward.

4 · Why top-down — the three justifications (recorded)

JustificationWhat it buys
Compliance by constructionNo deployment can produce non-standard content — misalignment is unrepresentable, not merely detectable. Governance is structural.
Governance by processCompliance is enforced at authoring time by the derivation rule, not by after-the-fact tracking of "which generated content aligns." Eliminates a perpetual audit burden and a class of silent gaps.
Inference-layer simplicity (decisive)Content is born structured in a canonical shape, so the AI agent layer (J2) maps into a known structure instead of reconstructing structure at inference time. Structuring cost is paid once at authoring rather than every time at inference — a compounding advantage for an inference-heavy agentic OS. This is the strongest reason the taxonomy must be locked before J2 is built.

5 · Governed growth — how the superset expands

The superset is dynamic; it must grow without breaking compliance-by-construction. Growth is therefore itself a governed, provenance-sealed process, not an ad-hoc edit.

  • Growth by immutable versioned extension. New knowledge / revised research / evolving pedagogy does not mutate existing nodes — it derives new versions (or new nodes), leaving prior versions intact with their provenance. Content authored against version N stays valid and traceable as the superset advances to N+1. This is exactly JUKOS's existing immutable-versioning discipline — the dynamism and the immutability are the same mechanism.
  • New domains as new branches. A new domain attaches by adding its standard-grounded (or structured-native) branch under the connective upper structure, through the same governed process.
  • Every extension is sealed and logged. Each addition to the superset is EDDNA-sealed (provenance of the extension) and Audit-Log-recorded (history of the change), so growth is traceable, governed, and IP-defensible. The superset can only grow through the governed path — which preserves the compliance guarantee as it expands.

The encouraging finding: the corpus already built the mechanism (immutable-versioning, EDDNA, audit, terminology layer, downward specialisation, adopter pattern) — with a different default posture (sovereign + optional map-outward). This spec does not discard that; it flips the posture to mandatory top-down derivation and adds two structural pieces (the connective unification frame + the governed extension process) plus a data task (populate the standards catalogue). Built work (J0, J1.1–J1.3) is compatible and needs augmentation, not rework.

7 · What is JUKOS-owned IP here

Two pieces are genuinely novel and constitute the defensible taxonomy IP:

  • The connective upper structure — unifying heterogeneous global standards (which do not share a common frame) into one coherent, derivable superset is a hard, valuable problem. This unification frame is JUKOS-owned and is the heart of "OS-level universal taxonomy."
  • The governed extension process — the disciplined, sealed, versioned mechanism by which the superset grows (new knowledge/research/pedagogy/domains) while preserving compliance-by-construction. A dynamic universal taxonomy that stays governed as it grows is the differentiator.

  1. Reconcile first (read-only). Confirm precisely how the corpus's UKG / EKG / terminology-layer / immutable-versioning machinery supports this spec — what already holds vs. the delta in §6 — with page cites. Report before building.

  3. Build the connective upper structure — the JUKOS-owned unification frame above the branches, sovereign-identity, standard-bound.
  4. Flip the derivation posture to mandatory-top-down — enforce, at mint, that every node derives downward from a superset parent and (where available) is standard-grounded, immutably recorded. Add the structured-native path for standard-less domains.
  5. Formalise the governed extension process — the sealed/versioned rules for admitting new knowledge/branches, on the existing EDDNA + Audit + immutable-versioning machinery.
  6. Preserve all standing invariants — additive on the jukos DB, p237 separation, inert to AVF (voice/GPU/managed-count untouched), OSAAT-gated, report + runbook + commit per step.

10 · Sequencing (how this aligns without stalling)

  1. This spec = the target (done — this document).
  2. Reconcile against the corpus (CC read-only recon) — the delta in §6, precisely, with cites.
  3. Revise the taxonomy layer to this spec (CC build, gated) — the posture flip + the two novel pieces + the catalogue.
  4. Resume forward — J1.4 proceeds in parallel now (it is taxonomy-independent: renderer adapter + P5), then J2 is built on the aligned superset, reaping the inference-simplicity benefit that justified this whole design.
Why lock this before J2. J2 (the agent/inference layer) is the primary beneficiary of born-structured content. Building J2 on the un-aligned (optional-map-outward) taxonomy and aligning afterward would force a rework of the agent layer. The taxonomy foundation is set (J0/J1.1–1.3) and the agent layer is not yet built — this is the cheapest, and nearly the last clean, moment to align. J1.4 does not depend on the taxonomy and need not wait.
